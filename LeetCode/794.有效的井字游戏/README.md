## 题目描述
用字符串数组作为井字游戏的游戏板 board。当且仅当在井字游戏过程中，玩家有可能将字符放置成游戏板所显示的状态时，才返回 true。

该游戏板是一个 3 x 3 数组，由字符 " "，"X" 和 "O" 组成。字符 " " 代表一个空位。

以下是井字游戏的规则：
```
玩家轮流将字符放入空位（" "）中。
第一个玩家总是放字符 “X”，且第二个玩家总是放字符 “O”。
“X” 和 “O” 只允许放置在空位中，不允许对已放有字符的位置进行填充。
当有 3 个相同（且非空）的字符填充任何行、列或对角线时，游戏结束。
当所有位置非空时，也算为游戏结束。
如果游戏结束，玩家不允许再放置字符。
```
示例 1:
```
输入: board = ["O  ", "   ", "   "]
输出: false
解释: 第一个玩家总是放置“X”。
```
示例 2:
```
输入: board = ["XOX", " X ", "   "]
输出: false
解释: 玩家应该是轮流放置的。
```
示例 3:
```
输入: board = ["XXX", "   ", "OOO"]
输出: false
```
示例 4:
```
输入: board = ["XOX", "O O", "XOX"]
输出: true
```
说明:
```
游戏板 board 是长度为 3 的字符串数组，其中每个字符串 board[i] 的长度为 3。
 board[i][j] 是集合 {" ", "X", "O"} 中的一个字符。
```

### 解法 模拟
一道面向用例编程的题目…
因为棋盘一共就9个格子，很明显不用什么技巧，直接rule-base干就完了。

然而判规则也需要一些突破口。
我的思路是先统计棋子个数。这是最显而易见的一个突破口。
因为两个玩家是轮流放棋的，且X先放，因此任意时刻，棋盘上X和O的个数只可能是相同或者X多一个。

在排除掉所有非上述两种情况之后，对于上述两种情况，就需要进一步查看棋盘的X和O的各自的成行数了。
此时进入分类讨论。

针对`counter['X'] == counter['O']`的情况，类似
```text
XXX
XOO
OO 
```
的例子是False，因为相同意味着最新下棋的是O，然而在那之前X已经成行了。
换言之，这种情况下，X的成行数必须是等于0的。否则永远不会有`counter['X'] == counter['O']`的情况出现。

另一种，当`counter['X'] == counter['O'] + 1`时，
和上述情况相反，此时应该保证O的成行数是0。

到这里其实整个判断逻辑链就完整了。
